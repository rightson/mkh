# 股癌市場心法 → 分時期方法論

把指定時期的股癌逐字稿交給 `gooaye-methodology-extractor` agent 平行抽取，
再綜合成一份該時期的方法論文件。

## 流程

1. **決定範圍**：
   - 參數是時期代號（如 `2025H1`）：查 `methodology/README.md` 的分期表取得集數範圍。
   - 參數是集數範圍（如 `EP476-EP525`）：直接使用。
   - 無參數：從 `methodology/README.md` 進度表挑最近一個未完成時期。
   - 用 `grep '^date:' transcripts/EPxxx.md` 抽查邊界集數日期，確認分期正確。

2. **平行抽取**：以 **5 集為一批**（單集約 3 萬 tokens，5 集接近 agent 上下文上限，
   勿加大批次），每批派一個 `gooaye-methodology-extractor` agent，
   prompt 寫明確切檔案清單（如 `transcripts/EP626.md` ... `transcripts/EP630.md`）。
   同一則訊息一次發出所有批次讓它們平行執行。

3. **綜合**：等全部 agent 回來後，把各批結果綜合成 `methodology/<時期>.md`，結構：
   - `# 股癌市場方法論 <時期>（EPxxx–EPyyy，日期範圍）`
   - `## 時期背景`：該半年的市場大事與他的整體立場
   - `## 核心心法`：依五大分類（市場判斷框架／操作原則／風控與倉位／情緒與心理／資訊處理）
     整理**跨集重複出現**的原則，每條附集數出處（如 `EP675, EP662`）
   - `## 該時期特有的戰術`：只適用當時情境的操作（標明情境條件）
   - `## 自我檢討錄`：他認錯與修正的案例（最有價值的部分，務必保留）
   - `## 金句選`
   - 綜合時去重、合併同義心法，但**保留集數出處**以便回查。

4. **更新進度**：在 `methodology/README.md` 的進度表把該時期標記完成、填入文件連結。

5. **交付**：commit（訊息格式：`新增 <時期> 市場方法論（EPxxx–EPyyy）`）。
   若使用者在遠端環境有指定分支，push 到該分支。

## 品質標準

- 心法必須有集數出處，不可泛泛而談。
- 「不同時期」是重點：綜合時注意他的方法隨市況演變（多頭追動能 vs 空頭保守），
  在「時期背景」中點出與前一時期文件的差異（若已存在前後時期文件，讀來對照）。
- 業配與閒聊不入文；自我檢討與認錯是高價值內容，優先保留。
